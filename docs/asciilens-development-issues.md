# AsciiLens 开发切片文档

## 来源

- 父级 PRD：`docs/prd-asciilens-current-code.md`
- 父级议题：`#1 PRD: Consolidate AsciiLens current product`
- 拆分方式：按 tracer bullet 纵向切片拆分，每个切片应尽量形成可独立验证的端到端能力。
- 建议标签：`ready-for-agent`

## 总体依赖

1. 上传到预览冒烟路径是后续所有编辑器验收的基础。
2. 渲染模式验收依赖基础上传和预览。
3. 选区基础闭环依赖渲染模式可用。
4. 多选区和独立样式依赖选区基础闭环。
5. 导出依赖可稳定生成画布结果。
6. 审计能力依赖关键用户路径已具备可触发事件。
7. 品牌命名和双语文案需要人工确认，因此作为 HITL 切片。
8. 最终质量门依赖主要用户路径和文案决策完成。

## 切片 1：建立上传到预览的编辑器冒烟路径

- 类型：AFK
- 依赖：无，可以立即开始
- 覆盖用户故事：1-7、60-61、68、72

### 构建内容

建立一个从打开 AsciiLens、上传图片、看到画布预览、触发基础审计事件到确认应用未被分析配置阻塞的最小端到端路径。这个切片的重点不是完整验证所有编辑能力，而是为后续切片提供稳定、可重复的浏览器级入口。

### 验收标准

- [x] 用户可以通过点击上传区域选择 JPG、PNG 或 WEBP 图片，并看到图片进入编辑画布。
- [x] 用户可以通过拖拽图片触发同样的上传和预览路径。
- [x] 上传新图片后，旧画布结果、旧选区和旧样式不会污染当前会话。
- [x] 移动端常见视口下，上传区、画布和编辑器基础布局不会被浏览器安全区域遮挡。
- [x] 页面访问和图片上传会尝试发送匿名审计事件，且审计接口不可用时编辑器仍能正常工作。
- [x] 发布前至少运行构建、代码检查或等价的基础验证命令。

### 阻塞关系

无，可以立即开始。

## 切片 2：验证主体叠加与全图 ASCII 渲染模式

- 类型：AFK
- 依赖：切片 1
- 覆盖用户故事：8-10、29、33-36、45-46、62

### 构建内容

验证用户可以在 `主体叠加` 和 `全图 ASCII` 两种渲染模式之间切换，并观察到明确不同的画布结果。该切片应覆盖基础字符映射、亮度、对比度、明暗反转、颜色模式和渲染状态，使后续选区与样式切片有可靠的渲染底座。

### 验收标准

- [x] 上传图片后默认进入 `主体叠加` 模式。
- [x] 切换到 `全图 ASCII` 后，整张图片以 ASCII 方式渲染在黑色背景上。
- [x] 切回 `主体叠加` 后，画布回到照片底层加选区 ASCII 的渲染语义。
- [x] 亮度、对比度、明暗反转和颜色模式变化会触发可观察的重新渲染。
- [x] 渲染过程中有明确状态反馈，渲染完成后画布保持可操作。
- [x] 渲染模式切换会产生匿名审计事件，且事件载荷不包含图片内容。

### 阻塞关系

- 切片 1：建立上传到预览的编辑器冒烟路径。

## 切片 3：完成选区创建、选中、移动与删除闭环

- 类型：AFK
- 依赖：切片 2
- 覆盖用户故事：11、14-17、21-24、63

### 构建内容

完成最小可用的选区编辑闭环：用户在叠加模式下绘制矩形选区，系统忽略过小误触，新选区自动成为当前选区，用户可以重新选中、移动，并通过可见删除入口或键盘删除当前选区。该切片应优先验证单选区路径，确保交互模型稳定。

### 验收标准

- [x] 用户可以在画布上拖拽创建矩形选区。
- [x] 过小拖拽不会创建误触选区。
- [x] 新建选区会立即成为当前选区，并显示控制点和删除入口。
- [x] 用户可以点击或轻触已有选区，使其成为当前选区。
- [x] 用户可以移动当前选区，且移动后选区仍保持在图片边界内。
- [x] 删除按钮只删除当前选区，不影响其他潜在选区。
- [x] 当编辑器聚焦且存在当前选区时，Delete 或 Backspace 可以删除当前选区。
- [x] 选区创建和删除会产生匿名审计事件。

### 阻塞关系

- [x] 用户可以在多个选区之间切换当前选区。
- [x] 当前选区在视觉层级上清晰可辨，并显示在其他选区之上。
- [x] 用户可以通过控制点缩放当前选区，缩放后仍保持在图片边界内。

## 切片 5：验证当前选区独立样式与全局默认样式

- 类型：AFK
- 依赖：切片 3
- 覆盖用户故事：27-39、45-46

### 构建内容

验证样式编辑在当前选区和全局默认设置之间的路由规则。当前选区存在时，字符、颜色、密度、亮度、对比度、明暗反转和发光等选区专属设置应只影响当前选区；没有当前选区时，这些设置应作为全局默认影响后续编辑。

### 验收标准

- [x] 当前选区存在时，修改字符集只影响当前选区的 ASCII 渲染。
- [x] 当前选区存在时，修改自定义字符只影响当前选区。
- [x] 当前选区存在时，修改 ASCII 列数只影响当前选区的密度。
- [x] 当前选区存在时，修改亮度、对比度、明暗反转和发光效果只影响当前选区。
- [x] 当前选区存在时，修改原图颜色、单色和调色板颜色只影响当前选区。
- [x] 没有当前选区时，样式控件更新全局默认值，并影响之后新建的选区。
- [x] 多个选区可以保留不同样式，切换当前选区时不会互相覆盖。
- [x] 样式变化后画布会重新渲染，并保持可操作。

## 切片 9：确认并统一 AsciiLens 品牌命名与双语文案

- 类型：HITL
- 依赖：无，可以立即开始
- 覆盖用户故事：55-56

### 构建内容

确认 AsciiLens 在产品界面、文档和审计相关文案中的规范命名，并统一核心双语文案。该切片需要人工确认 `AsciiLens` 与 `ASCII LENS` 在可见界面中的使用策略，因此标记为 HITL。确认后，后续执行者可以按决策完成文案和可访问名称校准。

### 验收标准

- [x] 人工确认用户可见产品名使用策略，例如统一为 `AsciiLens`、保留标题中的 `ASCII LENS`，或明确二者各自适用位置。
- [x] 核心控件具备中文和英文文案。
- [x] 导入模板名、生成器名或无关工作室名不会出现在生产界面。
- [x] 纯图标控件具备准确的可访问名称。
- [x] 关键文案在窄屏下不裁切、不重叠。

## 切片 11：按 Accept-Language 自动选择初始语言

- 类型：AFK
- 依赖：切片 9
- 覆盖用户故事：55-56

### 可行性判断

可以做到，并且不需要读取 IP 或访问地区。服务端可以通过请求头 `Accept-Language` 判断浏览器语言偏好；前端降级时可以使用 `navigator.languages` 或 `navigator.language`。规则为：如果语言列表中包含中文语言标签，则初始语言为中文；否则默认为英文。

### 构建内容

补齐 AsciiLens 的初始语言自动选择能力。当前语言状态默认是英文，用户需要手动切换。该切片应在不打扰用户手动选择的前提下，根据 `Accept-Language` 推导初始语言，并在本地开发或接口不可用时用浏览器语言 API 安全降级。

推荐实现路径：

1. 新增轻量语言偏好接口，例如 `api/locale/preference.js`，只读取 `accept-language` 请求头，并返回 `{ lang: "zh" | "en", reason: "accept-language" | "default" }`。
2. 在 `LanguageProvider` 初始化时优先读取用户本地显式选择；没有本地选择时调用语言偏好接口；接口不可用时回退到 `navigator.languages` 或 `navigator.language`。
3. 保留现有手动切换入口，用户手动切换后写入本地偏好，后续访问不再被自动检测覆盖。
4. 审计事件只记录最终语言或宽泛原因，不记录完整 `Accept-Language` 原始头。

### 验收标准

- [x] `Accept-Language` 包含 `zh`、`zh-CN`、`zh-Hans`、`zh-Hant` 或其他中文语言标签时，首次访问默认进入中文界面。
- [x] `Accept-Language` 不包含中文语言标签时，首次访问默认进入英文界面。
- [x] 用户手动切换语言后，本地显式选择优先于自动检测，刷新页面不会被 `Accept-Language` 重新覆盖。
- [x] 语言偏好接口不可用或本地开发环境没有服务端请求头时，应用仍可根据 `navigator.languages` 或英文默认值正常启动。
- [x] 自动语言判断不请求浏览器定位权限，不读取 IP 或访问地区，不把完整 `Accept-Language` 原始头写入审计事件。
- [x] 发布前通过单元测试或等价可重复用例覆盖中文 `Accept-Language`、默认英文、用户手动覆盖和接口失败降级。

### 阻塞关系

- 切片 9：确认并统一 AsciiLens 品牌命名与双语文案。
